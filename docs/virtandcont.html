Virtualization and Containerization

Before containerization we used to isolate applications in its own virtual machine.


Virtualization:

Virtualization is running applications in one physical hardware.

In virtualization you use a hypervisor between the host OS and the guess OS. it provides virtual hardware through these two. Basically, with virtualization you are able to import a guest operating system on top of a host operating system while allowing us to run multiple operating systems on different machines as they run on the same host.

We then shifted into containerization because virtualization had some cons that needed to be addressed

Containerization:

Containerization brings virtualization to the operating system. With this technique we do not use guest operating systems. Application binaries and libraries run on the host kernel which means that containerization is more convenient and time efficient. Docker containerizes a process or an application by giving it access to a kernel, but isolating it.

Similarities and differences:

Similarities:

  • With either technique you can run multiple applications inside a Host machine

Differences:

  • Containerization tends to be less heavy and faster than virtualization.
  • Containerization does not need a Guest Operating System
  • Virtualization can create many Operating systems in one host machine, while containerization will create multiple containers for each type of application.
